ALL API ENDPOINTS - WTPKP PRODUCTION

Base URL:
https://wtpkpapi.citasys.net

Panduan ini dibuat untuk tes via Postman.

Default headers untuk request JSON:
Accept: application/json
Content-Type: application/json

Header tambahan untuk endpoint yang butuh login:
Authorization: Bearer <token>

Urutan penggunaan paling umum:
1. Register user jika belum ada.
2. Login pakai username dan password.
3. Ambil token dari response login.
4. Pakai token di header Authorization untuk akses endpoint data.
5. Logout jika token sudah tidak dipakai.

Catatan penting:
- Login memakai username, bukan email.
- operator_name untuk data logsheet otomatis memakai full_name user yang sedang login.
- Jangan kirim operator_name saat create/update lewat API, kecuali benar-benar perlu override data lama.
- Response GET logsheet yang punya field lokasi akan menambahkan location_name dari master_locations. Input create/update tetap memakai lokasi sebagai kode lokasi.
- Input lokasi atau location_code pada logsheet harus memakai kode yang sudah ada di master_locations.
- Endpoint list selalu pagination.
- Query pagination umum: page dan per_page.
- Maksimal per_page adalah 100.
- Di Postman, pilih Body > raw > JSON untuk endpoint POST dan PUT.

2. REGISTER USER

Method:
POST

URL:
https://wtpkpapi.citasys.net/api/auth/register

Kegunaan:
Membuat user baru untuk login aplikasi/API.

Butuh token:
Tidak.

Headers:
Accept: application/json
Content-Type: application/json

Body raw JSON:
{
  "username": "operator",
  "password": "secret-password",
  "full_name": "Operator A",
  "email": "operator@example.com"
}

Rules:
- username wajib.
- password wajib, minimal 6 karakter.
- full_name wajib.
- email opsional.
- Jika email tidak dikirim, sistem membuat email internal otomatis.

7. LIST DATA RESOURCE

Method:
GET

URL pattern:
https://wtpkpapi.citasys.net/api/log-sheets/{resource}

Kegunaan:
Melihat data berdasarkan resource dengan pagination.

Butuh token:
Ya.

Headers:
Accept: application/json
Authorization: Bearer <token>

Params tab di Postman:
- page: nomor halaman, contoh 1.
- per_page: jumlah data per halaman, contoh 10.
- filter sesuai resource, contoh log_date, shift, lokasi, operator_name, location_code, unit_name, username, email, full_name.

Contoh URL list WTP:
https://wtpkpapi.citasys.net/api/log-sheets/wtp?page=1&per_page=10

Contoh URL filter WTP by lokasi:
https://wtpkpapi.citasys.net/api/log-sheets/wtp?per_page=10&lokasi=LOC001

Contoh URL filter Chemical Usage by tanggal:
https://wtpkpapi.citasys.net/api/log-sheets/chemical-usage?log_date=2026-05-13

Body:
Tidak perlu.

Catatan response:
Untuk resource yang punya field lokasi, response tetap menampilkan lokasi sebagai kode dan menambahkan location_name dari master_locations.

9. CREATE DATA RESOURCE

Method:
POST

URL pattern:
https://wtpkpapi.citasys.net/api/log-sheets/{resource}

Contoh URL:
https://wtpkpapi.citasys.net/api/log-sheets/wtp

Kegunaan:
Menambah data baru pada resource.

Butuh token:
Ya.

Headers:
Accept: application/json
Content-Type: application/json
Authorization: Bearer <token>

Catatan:
- Untuk logsheet, operator_name otomatis dari full_name user login.
- Body harus berisi field yang valid sesuai resource.
- Field shift menerima Shift 1, Shift 2, atau Shift 3. API menyimpan nilai tersebut sebagai Shift Pagi, Shift Siang, atau Shift Malam agar konsisten dengan data lama.
- Field lokasi dan location_code harus memakai kode yang sudah ada di master_locations.

Contoh Body raw JSON untuk WTP:
{
  "log_date": "2026-05-13 08:00:00",
  "shift": "Shift 1",
  "lokasi": "LOC001",
  "ph": 7.1,
  "cond": 12.3,
  "tds": 10.2,
  "sio2": 10.2
}

Contoh Body raw JSON untuk Chemical Usage:
{
  "log_date": "2026-05-13",
  "pac": 10.5,
  "poly": 2.25,
  "bio_pre": 1
}

- Field lokasi dan location_code harus memakai kode yang sudah ada di master_locations.

Contoh Body raw JSON:

RESOURCE YANG TERSEDIA

Format endpoint CRUD untuk semua resource di bawah:
- List:   GET    https://wtpkpapi.citasys.net/api/log-sheets/{resource}
- Detail: GET    https://wtpkpapi.citasys.net/api/log-sheets/{resource}/{id}
- Create: POST   https://wtpkpapi.citasys.net/api/log-sheets/{resource}
- Update: PUT    https://wtpkpapi.citasys.net/api/log-sheets/{resource}/{id}
- Delete: DELETE https://wtpkpapi.citasys.net/api/log-sheets/{resource}/{id}

Contoh jika resource adalah ro-process:

- PUT    https://wtpkpapi.citasys.net/api/log-sheets/ro-process/1
- DELETE https://wtpkpapi.citasys.net/api/log-sheets/ro-process/1

CONTOH CEPAT DI POSTMAN

Login:
Method: POST
URL: https://wtpkpapi.citasys.net/api/auth/login
Headers:
Accept: application/json
Content-Type: application/json
Body raw JSON:
{
  "username": "operator",
  "password": "secret-password"
}

Ambil data WTP:
Method: GET
URL: https://wtpkpapi.citasys.net/api/log-sheets/wtp?per_page=10
Headers:
Accept: application/json
Authorization: Bearer <token>
Body:
Tidak perlu.

Tambah data WTP:
Method: POST
URL: https://wtpkpapi.citasys.net/api/log-sheets/wtp
Headers:
Accept: application/json
Content-Type: application/json
Authorization: Bearer <token>
Catatan:
Nilai shift yang diterima: Shift 1, Shift 2, Shift 3.
Nilai shift yang disimpan: Shift Pagi, Shift Siang, Shift Malam.
Nilai lokasi harus memakai location_code yang sudah ada di master_locations.
Body raw JSON:
{
  "log_date": "2026-05-13 08:00:00",
  "shift": "Shift 1",
  "lokasi": "LOC001",
  "ph": 7.1,
  "cond": 12.3,
  "sio2": 10.2
}

Update data WTP:
Method: PUT
URL: https://wtpkpapi.citasys.net/api/log-sheets/wtp/1
Headers:
Accept: application/json
Content-Type: application/json
Authorization: Bearer <token>
Body raw JSON:
{
  "shift": "Shift 2",
  "ph": 7.2
}
