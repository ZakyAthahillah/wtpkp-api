ALL API ENDPOINTS - WTPKP

Headers used by JSON endpoints:
Accept: application/json
Content-Type: application/json

Headers used by protected endpoints:
Accept: application/json
Content-Type: application/json
Authorization: Bearer <token>

1. Health Check
GET /api/health

Example:
curl -X GET "http://localhost:8000/api/health" \
  -H "Accept: application/json"


2. Register
POST /api/auth/register

Body:
{
  "username": "operator",
  "password": "secret-password",
  "full_name": "Operator A",
  "email": "operator@example.com"
}

Notes:
- username wajib dan dipakai untuk login.
- password minimal 6 karakter.
- full_name wajib.
- email opsional. Jika tidak dikirim, sistem membuat email internal otomatis: <username>@wtpkp.local.

Example:
curl -X POST "http://localhost:8000/api/auth/register" \
  -H "Accept: application/json" \
  -H "Content-Type: application/json" \
  -d "{\"username\":\"operator\",\"password\":\"secret-password\",\"full_name\":\"Operator A\",\"email\":\"operator@example.com\"}"


3. Login
POST /api/auth/login

Body:
{
  "username": "operator",
  "password": "secret-password"
}

Example:
curl -X POST "http://localhost:8000/api/auth/login" \
  -H "Accept: application/json" \
  -H "Content-Type: application/json" \
  -d "{\"username\":\"operator\",\"password\":\"secret-password\"}"


4. Current User
GET /api/auth/me

Example:
curl -X GET "http://localhost:8000/api/auth/me" \
  -H "Accept: application/json" \
  -H "Authorization: Bearer <token>"


5. Logout
POST /api/auth/logout

Example:
curl -X POST "http://localhost:8000/api/auth/logout" \
  -H "Accept: application/json" \
  -H "Authorization: Bearer <token>"


6. Resource List
GET /api/resources

Example:
curl -X GET "http://localhost:8000/api/resources" \
  -H "Accept: application/json" \
  -H "Authorization: Bearer <token>"


7. Paginated List Data
GET /api/log-sheets/{resource}

Supported query params:
- page: page number
- per_page: rows per page, max 100
- resource-specific filters, for example log_date, lokasi, operator_name, location_code, unit_name, email, full_name

Example:
curl -X GET "http://localhost:8000/api/log-sheets/wtp?per_page=10&lokasi=LOC001" \
  -H "Accept: application/json" \
  -H "Authorization: Bearer <token>"

Example:
curl -X GET "http://localhost:8000/api/log-sheets/chemical-usage?log_date=2026-05-13" \
  -H "Accept: application/json" \
  -H "Authorization: Bearer <token>"


8. Show Detail Data
GET /api/log-sheets/{resource}/{id}

Example:
curl -X GET "http://localhost:8000/api/log-sheets/wtp/1" \
  -H "Accept: application/json" \
  -H "Authorization: Bearer <token>"


9. Create Data
POST /api/log-sheets/{resource}

Example WTP body:
{
  "log_date": "2026-05-13 08:00:00",
  "shift": "Shift 1",
  "lokasi": "LOC001",
  "ph": 7.123,
  "cond": 12.345,
  "tds": 10.25,
  "p_alk": 1.2,
  "m_alk": 2.3,
  "th": 3.4,
  "sio2": 4.5
}

Example:
curl -X POST "http://localhost:8000/api/log-sheets/wtp" \
  -H "Accept: application/json" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d "{\"lokasi\":\"LOC001\",\"ph\":7.1,\"cond\":12.3}"

Example Chemical Usage body:
{
  "log_date": "2026-05-13",
  "pac": 10.500,
  "poly": 2.250,
  "bio_pre": 1.000
}

Example:
curl -X POST "http://localhost:8000/api/log-sheets/chemical-usage" \
  -H "Accept: application/json" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d "{\"pac\":10.5,\"poly\":2.25}"


10. Update Data
PUT /api/log-sheets/{resource}/{id}

At least one valid field must be provided in the request body.

Example:
curl -X PUT "http://localhost:8000/api/log-sheets/wtp/1" \
  -H "Accept: application/json" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d "{\"ph\":7.2}"


11. Delete Data
DELETE /api/log-sheets/{resource}/{id}

Example:
curl -X DELETE "http://localhost:8000/api/log-sheets/wtp/1" \
  -H "Accept: application/json" \
  -H "Authorization: Bearer <token>"
