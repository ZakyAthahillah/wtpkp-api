WTPKP Log Sheets API

Base URL:
- /api

Authentication:
- POST /api/auth/register is public and creates a user with username/password.
- POST /api/auth/login is public.
- Login uses username and password.
- Protected endpoints require Authorization: Bearer <token>.
- POST /api/auth/logout revokes the active JWT until it expires.
- For log sheet create/update endpoints, operator_name is filled automatically from the logged-in user's full_name.
- For resources with shift, API accepts Shift 1, Shift 2, and Shift 3, then stores them as Shift Pagi, Shift Siang, and Shift Malam.
- GET list/detail responses for resources with lokasi include location_name from master_locations. Create/update still uses lokasi as the location code.
- Log sheet inputs for lokasi or location_code must match an existing master_locations.location_code.
- GET /api/masters/locations, GET /api/masters/standard-ro, and GET /api/masters/standard-ro-process return master references for log sheets.

Standard response:
- success: boolean
- message: string
- data: object|array|null
- meta: object|null
- errors: object|null

Resources:
- allowed-domains
- roles
- users
- user-auth
- master-locations
- master-standard-ro
- master-standard-ro-process
- wtp
- ro-plant

- wwtp-ol
- chemical-usage

- condenser
- mb-running-data
- ro-pump-running-hours
- swro-uf-pressure
- bwro-pressure
- water-production

Notes:
- List endpoints are paginated.
- POST and PUT are validated before database writes.
- Tables are PostgreSQL-friendly snake_case versions of the SQL Server schema in databasetable.txt.
- Configure the database through .env using DB_CONNECTION=pgsql and DB_DATABASE=wtpkp.
